# Composition Patterns & Originality Rules

## Basic Structure Types

Pick one structure type per image — don't mix too many.

### Workflow

Suits: input -> process -> output, content production, AI workflows, automation chains.

How to draw: input on the left, the ink character or a strange machine processing in the middle, output on the right, orange arrows for main flow direction.

### System Fragment

Suits: information sources, filters, databases, renderers, agent system internals.

How to draw: only 3-5 core modules, the character participates in one key action.

### Before/After Contrast

Suits: chaos/order, manual/automated, scattered/consolidated, anxious/stable.

How to draw: chaos on the left, stability on the right, orange arrow in the middle. Characters can be more exaggerated.

### Role/State

Suits: user pain points, creator states, too many tools, information anxiety, stuck-to-flowing transitions.

How to draw: 2-4 small states, each with a short annotation.

### Concept Metaphor

Suits: content factory, information warehouse, brain black-box, workflow machine, automated daily report.

How to draw: one large strange object or machine, minimal inputs, one output. Must be memorable.

### Method Layers

Suits: methodology frameworks, system tiers, capability stacks, content system layers.

How to draw: stacked boxes layer by layer — no formal pyramids; the character nearby hauling bricks or building.

### Map/Route

Suits: idea to launch, user journeys, content handoff paths, learning roadmaps.

How to draw: one winding path, a few nodes, the character leading the way or walking along.

### Mini Comic Panels

Suits: failure to success, real process, complaints, before/after usage changes.

How to draw: 2-4 small scenes, each panel expressing only one action.

## Original Metaphor Generation Method

Every time, invent a new metaphor from the current content — never copy old images.

### Three Steps

1. Replace the abstract concept with a physical action: stuck, leaking, getting heavy, sorting, settling, fermenting, opening a door, folding, unpacking, flowing back.
2. Replace the system structure with a low-tech object: broken machine, cardboard box, drawer, pipe, mailbox, strange gauge, scale, well, ladder, strange workstation.
3. Make the ink character perform the action: not standing beside it, but stuck in the machine, pulling the wrong wire, guarding a door, carrying, patching, weighing, holding a ladder, recording, stuffing something into a strange device.

### Available Object Pool

- Cardboard boxes, drawers, old machines, funnels, scales, mailboxes, doors, wells, ladders, pipes, yarn balls, gates, turntables, black boxes, hole punchers, pasta machines, clotheslines, strange workstations.
- Pick only 1-2 per image — don't pile them up.

### Character Action Pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, dismantle, label, recycle.
- Actions must serve the core meaning — don't be weird just for weirdness.

## Anti-Copying Rules

Never default-replicate prior compositions. Each illustration must invent a fresh visual metaphor from the specific content being illustrated.

Same topic must use a new metaphor. For example: "handoff path" doesn't have to be drawn as a route — could be the character connecting a content tail to a door handle; "one-source-many-uses" doesn't have to be a fish — could be the character pressing one paper ball into several shapes.
